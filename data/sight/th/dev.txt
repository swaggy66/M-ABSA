"ตัวแปรเสมือน") g : (x,y) --> g(x,y) = 2x+y จากวิธีการพิจารณาเชิงฟังก์ชัน (หรืออัลกอริทึม) ที่เข้มงวดนี้ จะเห็นชัดว่า f และ g เป็นอัลกอริทึมที่แตกต่างกัน นั่นคือฟังก์ชันที่แตกต่างกัน! เมื่ออธิบายสิ่งต่างๆ ด้วยวิธีนี้ ความสับสน อันตราย ความขัดแย้ง... ก็จะหมดไป: (df/dx)y = 1 และ (dg/du)v=2 ไม่มี pb อีกต่อไป!...นอกจากนี้ ตอนนี้จงตระหนักไว้อย่างชัดเจนว่าเราสามารถเขียนด้วยความเข้มงวดอย่างแท้จริง: (dg/du)v = (dg/dx)y แต่เฉพาะเมื่อคุณอ่านอัตลักษณ์นี้ด้วยวิธีที่ถูกต้องเท่านั้นที่ควรอ่าน นั่นคือผ่าน "จิตวิญญาณแห่งฟังก์ชัน" และไม่ใช่ "จิตวิญญาณแห่งตัวเลข" เหตุใดจึงเป็นเช่นนั้น? เนื่องจากตัวแปรในนิพจน์เหล่านี้ เป็นและต้องถูกมองว่าเป็น "ตัวแปรเสมือน"! พูดให้ชัดเจนขึ้นก็คือ นิพจน์ทั้งสอง (dg/du)v หรือ (dg/dx)y มีความหมายอย่างแท้จริง คือ กำหนดให้ตัวแปรตัวแรก (ของคู่ลำดับของผลคูณไขว้ของเซต) มีค่าคงที่ ดังนั้น เราจึงเขียน (dg/dk)@ (หรือ "ชื่อ" อื่นๆ ของตัวแปร) ได้อย่างเท่าเทียมกัน หากเราได้กำหนดฟังก์ชัน "ในแผนภูมิ" ดังกล่าว โดยกำหนดป้ายกำกับ (k,@) ด้วย :g : (k , @) --> 2k+@ นี่เป็นวิธีเดียวที่จะอธิบายสิ่งต่างๆ ได้อย่างชัดเจนและเคร่งครัด โดยไม่ก่อให้เกิดความสับสนหรือความขัดแย้งทางสัญกรณ์ใดๆ! เพียงแค่เรียกฟังก์ชันว่าฟังก์ชัน ไม่ใช่ "แมวหรือหมา" เล่นกับคำและแนวคิดผ่านสัญลักษณ์ที่ดู "โง่เขลา" ของ "นักฟิสิกส์" ซึ่งบางครั้งกลายเป็นนิสัย "ที่ไม่ดี" นั่นเอง อันที่จริงแล้วนี่คือความผิดพลาดของกฎลูกโซ่ที่เกิดขึ้นเมื่อมีคนระบุ f ว่าเป็น g อย่างเข้าใจผิด ถ้าคนเราต้องการแยกความแตกต่างตามตัวแปรเดียวกัน เช่น x โดยให้ y คงที่ ก็ทำได้! แต่เพื่อที่จะทำอย่างนั้นโดยไม่เลอะเทอะและผิดพลาด เราไม่ควรเปรียบเทียบ (df/dx)y=1 กับ (dg/du)/v=2 อย่างไร้เดียงสา เพราะทั้งสองต่างกันอย่างชัดเจน (และไม่มีอะไรผิดเลย!) แต่ควรคิดแบบ "เชิงฟังก์ชัน" แทน "เชิงตัวเลข" นั่นหมายความว่าเราต้องคำนวณ "ฟังก์ชันลูกโซ่" G อย่างชัดเจน ซึ่งแตกต่างจาก g แต่เกี่ยวข้องกับ g โดยการเปลี่ยนแปลงของตัวแปร ดังนั้น ก่อนอื่น เราจึงแสดงการเปลี่ยนแปลงของตัวแปรในวิธีที่ถูกต้อง! นั่นคือ ฟังก์ชันต่อไปนี้ :C : (x,y) --> C(x,y) = (u,v) = (x, y - x) ดังนั้น เราจึงจำได้ว่าฟังก์ชัน g ถูกกำหนดเป็นอัลกอริทึมต่อไปนี้ :g : (@,#) --> 2@ +# หรือเทียบเท่ากัน (เนื่องจากฟังก์ชันใดๆ ก็ตามที่เป็นอย่างแท้จริง คือ ฟังก์ชันนั้นกำลังกำหนดอัลกอริทึมอยู่!) แต่สะดวกกว่า (สำหรับเป้าหมายของการเชื่อมฟังก์ชัน) :g : (u,v) --> g(u,v) = 2u+v ฟังก์ชันทั้งสองนี้ถูกเชื่อมอย่างชาญฉลาดเพื่อสร้างฟังก์ชัน G ที่กำหนดดังต่อไปนี้ :G(x,y) = g(u,v) = g( (u,v) ) = g(C(x,y)) = (goC)( (x,y) ) = (goC)(x,y)(x,y) ตอนนี้เราเห็นอย่างชัดเจนว่าฟังก์ชัน G คืออะไร : ฟังก์ชันเชื่อม ของ g และ C :G = goC โดยที่ "o" ที่อยู่ตรงกลางหมายถึงการดำเนินการเชื่อมโยง (องค์ประกอบ) บนพื้นที่ฟังก์ชัน ซึ่งกำหนดโดยทั่วไปสำหรับฟังก์ชันสองฟังก์ชัน A : E -> F และ B : F -> G อย่างง่ายๆ คือ :BoA : E -> G , x --> BoA(x) = B(A(x)) ซึ่งอาจเรียกอีกอย่างว่า "กฎของ BOA" แทนที่จะใช้ "หมอก" และ "gof" ตามปกติของ "การเชื่อมโยงฟังก์ชัน"... (ล้อเล่นนะ!) ในตัวอย่างของเรา ในที่สุด เราจะได้ เนื่องจาก u=x และ v=y-x :G(x,y) = (goC)(x,y) = g(C(x,y)) = g( (u,v) ) = g(u,v) = g(x, y-x) = 2x + (y-x) = x +y ดังนั้นเราจึง "ค้นพบ" ว่า สำหรับ x ทั้งหมดและ y ทั้งหมด เราได้ :G(x,y) = f(x,y) นั่นหมายความว่าความเท่าเทียมเชิงฟังก์ชัน :G=f แล้วเราทำอะไรไป? หันกลับมา? ใช่ ใน sens เพราะเราได้ยืนกรานที่จะแยกความแตกต่างระหว่าง g กับ x ใน NUMERICAL SENSE แทนที่จะทำในลักษณะที่เหมาะสมเชิงฟังก์ชัน นั่นคือ แยกความแตกต่างตามตัวแปรแรกในอัลกอริธึม g : (@,#) --> g(@,#) = 2@ + # ดังนั้น ใน NUMERICAL SENSE นี้ x ไม่ใช่ตัวแปรเสมือนอีกต่อไป แต่เป็น "ตัวเลขที่แนบกับ f"! นักฟิสิกส์จะเรียกมันว่า "สิ่งที่สังเกตได้ทางฟิสิกส์" การมองสิ่งต่างๆ ในลักษณะนี้ทำให้เกิดความซับซ้อนและอาจนำไปสู่ความสับสนได้ หากไม่เข้าใจฟังก์ชัน CHAIN KEY และอันที่จริงแล้ว การแมปแผนภูมิท่อร่วม (จึงเรียกว่าการเปลี่ยนแปลงของแผนภูมิ ซึ่งมักจะหาอนุพันธ์ได้สำหรับท่อร่วมปกติ) ที่อยู่ภายใต้ทั้งหมดนี้... ดังนั้น ในความหมายหนึ่ง เรา "กลับด้าน" ซึ่งนำไปสู่ G=f! แต่ถึงกระนั้น เราก็ได้เขียนสิ่งต่างๆ อย่างชัดเจนในฟังก์ชันลูกโซ่ด้วยวิธีที่ถูกต้อง ซึ่งเผยให้เห็นว่ามีประโยชน์อย่างยิ่งในเรขาคณิตเชิงอนุพันธ์ขั้นสูงและนามธรรมมากขึ้น ซึ่งแนวคิดสำคัญของการครอบคลุมและการครอบคลุมสากล พื้นที่สัมผัส กลุ่ม พีชคณิตลี อนุพันธ์ลี อนุพันธ์โควาเรียนต์ การเชื่อมโยง รูปแบบเชิงอนุพันธ์ เป็นต้น ล้วนเกิดขึ้นอย่างมีประโยชน์ ซึ่งจำเป็นต้องมีการทำความเข้าใจและเชี่ยวชาญกระบวนการฟังก์ชันลูกโซ่ให้ชัดเจน! ในที่สุด บ่อยครั้ง การใช้สัญลักษณ์ในทางที่ผิดซึ่งกำหนด f=g อย่างเข้าใจผิด จะได้รับการจัดการโดยไม่เกิดความเสียหายจาก "คอมพิวเตอร์มนุษย์" ด้วยนิสัยและความคล่องตัวบางประการ แต่ในพื้นที่ที่ค่อนข้างง่ายเท่านั้น ในทางกลับกัน มันจะนำไปสู่ความสับสน ความเข้าใจผิด ผลลัพธ์ที่ผิดพลาดอย่างร้ายแรง และการตีความที่ไม่สมเหตุสมผลที่ผิดพลาด ในพื้นที่ที่ละเอียดกว่า... เหมือนกัน ไม่ใช่เพื่อตั้งชื่อพวกมัน ทฤษฎีสัมพันธภาพทั่วไป เรขาคณิตเชิงอนุพันธ์ของรีมาเนียน สสารควบแน่น พลาสมา ฯลฯ การดำเนินการฟังก์ชันแบบเชื่อมโยงเป็นหนึ่งในสิ่งที่ง่ายที่สุด แต่สำคัญที่สุดในคณิตศาสตร์ทั้งหมด! มันเป็นหน่วยการสร้างของโครงสร้างที่ซ้อนกัน แต่ละอันอยู่ภายในอันอื่นๆ เหมือนกัน กล่องแพนโดร่า แต่เป็น "กล่อง" ที่เปิดเผยโครงสร้างที่น่าสนใจ เช่น กลุ่ม พีชคณิต คลิฟฟอร์ด... โทโพส... : จุดฟังก์ชัน จากจุดไปยังจุด ฟังก์ชัน (หรือตัวดำเนินการ) จากฟังก์ชันไปยังฟังก์ชัน ฯลฯ ทั้งหมดรวมอยู่ใน... หมวดหมู่ Groetendick SPIRIT...deap!####[]
